PrintingTools NG 개요

PrintingTools NG를 사용하면 이메일 인쇄에 대한 다양한 사용자 정의가 가능합니다. 확장에는 헤더와 메시지 부분 모두에 대한 많은 옵션이 있습니다. 또한 일반 Thunderbird 인쇄 미리보기에서는 수행할 수 없는 페이지 옵션을 설정할 수 있습니다. 유연한 토큰 기반 시스템을 사용하여 자동 이름 지정으로 PDF 출력을 수행할 수도 있습니다.

메일 헤더 도구 모음에 인쇄 NG 버튼 추가:

Thunderbird v115부터 통합 툴바에 Print NG 버튼이 자동으로 추가되었습니다. 메일 헤더 도구 모음에 버튼을 추가하여 인쇄 액세스와 메일 창의 옵션을 모두 활성화할 수도 있습니다. 또한 애드온 탭의 확장 기능에 대한 렌치 버튼을 통해 옵션에 액세스할 수 있습니다.


메일 헤더 도구 모음에 버튼을 배치하려면 다음 단계를 따르세요.

  • 메일 헤더 도구 모음에서 더보기 버튼을 클릭하세요.
  • 사용자 지정...을 선택하면 도구 모음 사용자 지정 창이 열립니다.
  • 인쇄 NG 버튼을 찾으세요
  • 버튼을 원하는 위치로 드래그 앤 드롭하세요.
  • 인쇄, 미리보기, 옵션 및 도움말 버튼을 사용하세요.

    • PrintingTools NG 키보드 단축키

    PrintingTools는 표준 Thunderbird 인쇄 바로가기를 다시 매핑하여 표준 인쇄 대신 자체적으로 호출합니다. Windows 및 Linux에서는 Mac에서 Control-P 및 Command-P를 사용하세요. Control-Shift-P를 사용하여 옵션 대화 상자를 열 수도 있습니다.

    다음은 헤더 형식 지정 기능 중 일부의 예입니다.

  • CSS를 사용하여 사용자 정의된 헤더 테두리 스타일(상단 및 하단, 파란색)
  • 헤더 배경색
  • 정렬된 헤더
  • 사용자 정의된 헤더 순서
  • 페이지 제목 및 메시지 폴더
  • 헤더에 아이콘이 있는 첨부 파일 목록

    • 사용자 정의 날짜 형식

    사용자 정의 날짜 옵션을 사용하면 이메일 헤더의 날짜 및 수신 날짜 모두에 사용되는 날짜 형식을 완벽하게 제어할 수 있습니다. 사용자 정의 날짜는 거의 모든 형식을 허용하는 여러 현지화된 옵션을 제공합니다. 사용자 정의 날짜 문자열에는 아래 나열된 날짜\시간 토큰과 혼합된 유니코드 문자를 포함한 리터럴, 구분 기호가 포함될 수 있습니다.

    참고: 아래 예의 사용자 정의 날짜 결과는 실제 날짜, 사용자 로케일 및 시간대에 따라 다릅니다.


    예:

    날짜 문자열 결과 메모
    %Y%0%m%1%d%2 %a %t %H:%M:%S [GMT %z] 2022年04月20日 水 JST 16:06:45 [GMT +0900] ja 리터럴이 포함된 로케일
    %Y%0%m%1%d%2 %a %t %H:%M:%S 2022년04월20일 수 KST 16:11:06 리터럴이 있는 ko 로케일
    %Y%0%m%1%d%2 %a %H:%M:%S %Z 2022年04月20日 周三 18:01:42 北京时间 zh-CN 로케일, 짧은 날
    %Y%0%m%1%d%2 %A %H:%M:%S %t 2022年04月20日 星期三 18:01:42 CST zh-TW 로케일, 긴 하루
    %d.%m.%Y %H:%M:%S %t 20.04.2022 18:01:42 CET 점 구분 기호
    %a, %b %d %Y %H:%M:%S Mi, Apr 20 2022 18:01:42 지역별, 짧은 일/월
    %A, %B %d %Y (%H:%M:%S) %t Wednesday, April 20 2022 (18:01:42) EDT en-US 로케일, 긴 일/월

    사용자 정의 날짜 형식은 기능과 유사한 strftime() Unix와 일치하는 % 시퀀스를 활용합니다. 날짜와 시간 형식을 지정하는 함수인 strftime() 의 JavaScript 포트를 기반으로 합니다.
    TH Doan의 strftime() 포트(https://thdoan.github.io/strftime/)


    변환 순서 설명

    순서 설명
    %a 요일의 약칭.
    %A 요일의 전체 이름입니다.
    %b 약식 월 이름입니다.
    %B 전체 월 이름입니다.
    %c 현재 로케일에 대한 기본 날짜 및 시간(UTC) 표현입니다.
    %C 2자리 정수로 된 세기 수(연도/100)입니다.
    %d 10진수로 표시된 날짜(범위 01~31)입니다.
    %e 10진수로 표시된 날짜(범위 1~31)입니다.
    %F ISO 8601 날짜 형식( %Y-%m-%d 와 동일)
    %G ISO 8601 십진수로 세기를 포함한 주 기반 연도입니다. 4자리 연도는 ISO 주 번호에 해당합니다( %V 참조). ISO 주 번호가 이전 또는 다음 연도에 속하는 경우 해당 연도가 대신 사용된다는 점을 제외하면 %Y 와 형식 및 값이 동일합니다.
    %g %G 비슷하지만 세기가 없습니다. 즉, 2자리 연도(00-99)가 있습니다.
    %H 24시간제(범위 00~23)를 사용하여 십진수로 시간을 표시합니다. %k 도 참조하세요.
    %I 12시간제를 사용하여 10진수로 시간을 표시합니다(범위 01~12). %l 도 참조하세요.
    %j 10진수로 표시된 연도의 일(범위 001~366)입니다.
    %k 24시간제(범위 0~23)를 사용하여 십진수로 시간을 표시합니다. %H 도 참조하세요.
    %l 12시간제를 사용하여 10진수로 시간을 표시합니다(범위 1~12). %I 도 참조하세요.
    %m 월을 십진수로 표시합니다(범위 01~12).
    %n 월을 십진수로 표시합니다(범위 1~12).
    %M 10진수로 표현된 분(범위 00~59)
    %p 주어진 시간 값에 따라 "AM" 또는 "PM"입니다. 정오는 "PM"으로 처리되고 자정은 "AM"으로 처리됩니다.
    %P %p 와 비슷하지만 소문자("am" 또는 "pm")입니다.
    %s Epoch 이후의 초 수, 1970-01-01 00:00:00 +0000(UTC).
    %S 10진수로 표현된 초(범위 00~59)
    %u 요일을 십진수(1~7 범위)로 표시하고 월요일은 1입니다. %w 도 참조하세요.
    %V ISO 8601 현재 연도의 주 번호를 십진수(범위 01~53)로 표시합니다. 여기서 1주는 새해에 최소 4일이 있는 첫 번째 주(즉, 첫 번째 목요일)입니다.
    %w 요일을 십진수(범위 0~6)로 표시하고 일요일은 0입니다. %u 도 참조하세요.
    %x 시간 없이 현재 로케일에 대한 기본 날짜 표현입니다.
    %X 날짜 없이 현재 로케일에 대한 기본 시간 표현입니다.
    %y 세기가 없는 십진수로 표현된 연도(범위 00~99).
    %Y 세기를 포함한 십진수로 표현된 연도입니다.
    %z +hhmm 또는 -hhmm 숫자 시간대(즉, UTC로부터의 시간 및 분 오프셋)입니다.
    %Z 시간대 이름 또는 약어입니다.
    %t 짧은 시간대 약어.
    %0 연도 리터럴. 주로 아시아 지역에 맞게 현지화되었습니다. (zh-CN: 年, ko: 년)
    %1 월 리터럴. 주로 아시아 지역에 맞게 현지화되었습니다. (zh-TW: 月, ko: 월)
    %2 일 리터럴. 주로 아시아 지역에 맞게 현지화되었습니다. (ja: 일, ko: 일)

    호환성 참고사항:

    • %c - 형식화된 문자열이 약간 다릅니다.
    • %D - 구현되지 않음( %m/%d/%y 또는 %d/%m/%y 사용)
    • %e - 공백이 추가되지 않았습니다.
    • %E - 구현되지 않음
    • %h - 구현되지 않음( %b 사용)
    • %k - 공백이 추가되지 않았습니다.
    • %n - %m 과 비슷하지만 앞에 0이 없습니다(개행에는 \n 사용).
    • %O - 구현되지 않음
    • %r - 구현되지 않음( %I:%M:%S %p 사용)
    • %R - 구현되지 않음( %H:%M 사용)
    • %t - 구현되지 않음( \t 사용)
    • %T - 구현되지 않음( %H:%M:%S 사용)
    • %U - 구현되지 않음
    • %W - 구현되지 않음
    • %+ - 구현되지 않음
    • %% - 구현되지 않음( % 사용)

    PDF 출력 및 자동 파일 이름 지정

    PDF 출력은 이제 토큰 시스템을 사용하여 자동 이름을 지정하고 사전 정의된 PDF 출력 디렉터리에 저장할 수 있습니다. 각 이메일의 이름과 위치를 선택할 필요가 없으므로 선택한 여러 이메일을 인쇄할 때 특히 유용합니다.

    PDF 출력 디렉토리를 활성화하고 저장 위치에 대한 디렉토리를 선택하십시오. 출력 파일 이름은 구성된 파일 이름 형식을 따릅니다.

    파일 이름 형식은 토큰과 구분 기호 또는 공백과 같은 정적 문자로 구성됩니다. 확장된 파일 이름 형식은 토큰을 활용합니다. 토큰과 텍스트 또는 유효한 파일 이름 문자를 활용하여 정확한 파일 이름을 구성할 수 있습니다. 잘못된 문자는 다음과 같습니다. | * : < > ? \ / "

    기본 이름 형식은 이메일 제목인 ${subject}입니다.

    예

    ${subject} - 기본 형식, 이메일 제목
    ${prefix}_${date_custom}_${sender} - 기본값 대신 사용자 정의 날짜 형식을 사용합니다.
    ${subject}-GmailMsg-${date}-${index} - 일부 맞춤 텍스트 포함

    아래 표에서는 파일 이름 형식에 사용할 수 있는 전체 토큰 세트를 설명합니다.

    토큰 설명
    ${subject} 메시지의 제목입니다.
    ${sender} 메시지 보낸 사람의 이름입니다.
    ${recipient} 메시지 수신자의 이름입니다.
    ${sender_email} 메시지 보낸 사람의 이메일 주소입니다.
    ${recipient_email} 메시지 수신자의 이메일 주소입니다.
    ${smart_name} 메시지가 보낸 편지함 폴더에 있는 경우 받는 사람, 그렇지 않은 경우 보낸 사람 이름
    ${index} 메시지 인덱스 번호입니다.
    ${prefix} 접두사 텍스트 옵션 값입니다.
    ${suffix} 접미사 텍스트 옵션 값입니다.
    ${date} 표준 날짜 형식입니다. 기본 형식은 YYYYMMDDHHmm입니다.
    ${date_custom} 사용자 정의 날짜 형식입니다. 사용자 정의 날짜 형식 섹션을 참조하세요.

    인쇄 및 페이지 옵션

    이 옵션을 사용하면 기본 프린터와 프린터 페이지별 옵션을 설정할 수 있습니다. 미리보기 대화 상자가 표시되지 않고 즉시 인쇄가 완료되는 무음 모드를 설정할 수도 있습니다.

    페이지 옵션을 사용하면 미리보기 또는 인쇄가 완료될 때 항목을 설정할 수 있습니다. 이러한 옵션은 인쇄 전반에 걸쳐 지속됩니다. PrintingTools NG에 설정된 옵션이 인쇄 미리보기에서 변경되어도 후속 인쇄에는 영향을 미치지 않습니다. 일부 항목은 미리보기 대화 상자에서 설정할 수 없으며 여기에는 머리글 및 바닥글이 포함됩니다. 다른 항목은 미리보기에 유지되지 않으며 일반적으로 인쇄가 완료될 때마다 설정해야 합니다. PrintingTools NG는 이 두 가지 문제를 모두 해결합니다.

    미리보기로 저장되지 않은 항목 중 하나는 페이지 범위입니다. 일반적인 사용 사례는 이메일의 첫 페이지만 기본값으로 인쇄하려는 경우입니다. 일반적으로 페이지 범위를 매번 변경해야 합니다. PrintingTools NG에 설정된 페이지 범위가 페이지 수를 초과하는 등 유효하지 않은 경우 인쇄하기 전에 미리보기에서 범위를 수정해야 합니다. "1"로 설정된 페이지 범위는 항상 유효하며 메시지의 첫 번째 페이지를 인쇄합니다.

    노트:

    스케일링 옵션을 사용하면 지속적인 스케일을 설정할 수 있습니다.

    페이지 범위는 미리보기와 동일한 방식으로 설명됩니다. 예는 1, 1-2, 3-4,5입니다. 미리 설정된 페이지 범위가 특정 메시지에 대해 유효하지 않은 경우 미리보기에 오류가 표시됩니다.

    여백은 사용자 로케일에 적합한 단위(인치 또는 밀리미터(in 또는 mm))를 사용하여 설정할 수 있습니다. 반올림 오류로 인해 일부 mm 값은 작은 차이로 저장될 수 있습니다.

    머리글과 바닥글은 아래 나열된 다양한 메시지 변수를 포함하도록 설정할 수 있습니다. 추가적으로 임의의 텍스트도 포함될 수 있습니다. 머리글이나 바닥글을 인쇄하지 않고 인쇄 공간을 절약하려면 모든 필드를 비워두세요.

  • &D = 날짜
  • &P = 페이지 번호
  • &PT = 총 페이지 번호(“3/5”)
  • &T = 페이지 제목(메시지 제목)
  • &U = URL(메시지의 URL)

    • 고급 옵션

    기본 UI 설정 외에 인쇄 항목을 설정할 수 있는 옵션입니다. 현재 한 가지 고급 옵션은 머리글과 바닥글의 가장자리 여백을 제어하는 것입니다. 옵션은 기타 탭의 고급 옵션에서 설정되며 공백으로 구분됩니다.
    형식은 다음과 같습니다.

    P: 프린터 이름 ::S: 옵션 이름 = 값

    프린터 이름에는 밑줄로 대체된 공백이 있으며 내장된 PDF에 저장은 Mozilla_Save_to_PDF입니다.

    또 다른 예: Microsoft_Print_to_PDF

    현재 옵션 이름 가능성은 다음과 같습니다.
    edges, edgeTop, edgeBottom, edgeLeft, edgeRight

    edges 모든 가장자리 여백을 동일한 값으로 설정하는 의사 옵션입니다. 가장자리 여백의 단위는 인치입니다.

    예:

    P:Mozilla_Save_to_PDF::S:가장자리=0.25
    P:Microsoft_Print_to_PDF::S:edgeTop=0.1

    edgeTop을 0.25인치로 제외한 모든 가장자리 여백을 0.10인치로 설정하려면 다음을 입력하십시오.

    P:Mozilla_Save_to_PDF::S:edges=0.10 P:Mozilla_Save_to_PDF::S:edgeTop=0.25

    디버그 옵션

    디버그 옵션은 공백으로 구분된 목록으로 입력되는 키워드 세트입니다. 옵션은 일반적으로 콘솔 디버깅 출력에 사용되지만 테스트 또는 다른 모드에도 사용될 수도 있습니다.

  • initialsource - PTNG가 형식을 지정하기 전에 메시지 소스를 표시합니다.
  • finaloutput - 최종 메시지 출력을 표시합니다.
  • pdfoutput - PDF 메시지 출력에 대한 디버그 출력
  • printsettings - 인쇄 및 페이지 옵션에 대한 디버그 출력
  • helpinwin - 탭의 표준 대신 창에 도움말을 표시하는 모드 플래그

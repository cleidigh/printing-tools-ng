PrintingTools NG Översikt

PrintingTools NG möjliggör många anpassningar av e-postutskrift. Tillägget har många alternativ för både rubrikerna och meddelandedelarna. Dessutom kan sidalternativ ställas in som inte kan göras i den normala Thunderbird-förhandsgranskningen. PDF-utdata kan också göras med automatisk namngivning med ett flexibelt tokenbaserat system.

Lägga till knappen Print NG i verktygsfältet:

Knappen Skriv ut NG bör läggas till i verktygsfältet för att möjliggöra både utskriftsåtkomst och alternativ. Ytterligare alternativ kan nås med skiftnyckelknappen för tillägget på tilläggsfliken.


Följ dessa steg för att placera knappen i verktygsfältet:

  • Högerklicka på ett tomt utrymme i e-postverktygsfältet
  • Välj Anpassad... Detta öppnar fönstret Anpassa verktygsfält
  • Leta upp knappen Skriv ut NG
  • Dra och släpp knappen till önskad plats
  • Använd knappen för Skriv ut, Förhandsgranska, Alternativ och Hjälp

    • Följande är ett exempel på några av headerformateringsfunktionerna:

  • Anpassad rubrikkantsstil med CSS (överst och nederst, blå)
  • Rubrik bakgrundsfärg
  • Justerade rubriker
  • Anpassad rubrikordning
  • Sidrubrik och meddelandemapp
  • Bilaga lista med ikoner i rubriken

    • Anpassat datumformat

    Alternativet anpassat datum tillåter fullständig kontroll över datumformateringen som används för både datum och mottaget datum i e-posthuvudena. Det anpassade datumet ger flera lokaliserade alternativ som tillåter nästan alla format. Observera att den anpassade datumsträngen kan inkludera separatorer, bokstaver inklusive unicode-tecken blandade med datum/tid-tokens som anges nedan.

    Obs! De anpassade datumresultaten i exemplen nedan kommer att skilja sig beroende på det faktiska datumet, användarens språk och tidszon.


    Exempel:

    Datumsträng Resultat Notera
    %Y%0%m%1%d%2 %a %t %H:%M:%S [GMT %z] 2022年04月20日 水 JST 16:06:45 [GMT +0900] ja lokalitet med bokstaver
    %Y%0%m%1%d%2 %a %t %H:%M:%S 2022년04월20일 수 KST 16:11:06 ko-lokal med bokstaver
    %Y%0%m%1%d%2 %a %H:%M:%S %Z 2022年04月20日 周三 18:01:42 北京时间 zh-CN-lokal, kortdag
    %Y%0%m%1%d%2 %A %H:%M:%S %t 2022年04月20日 星期三 18:01:42 CST zh-TW-lokal, lång dag
    %d.%m.%Y %H:%M:%S %t 20.04.2022 18:01:42 CET Punktavskiljare
    %a, %b %d %Y %H:%M:%S Mi, Apr 20 2022 18:01:42 de locale, kort dag/månad
    %A, %B %d %Y (%H:%M:%S) %t Wednesday, April 20 2022 (18:01:42) EDT en-US språk, lång dag/månad

    Det anpassade datumformatet använder %-sekvenser som matchar den strftime() Unix-liknande funktionen. Den är baserad på JavaScript-porten strftime() , en funktion för att formatera datum och tid.
    Port of strftime() av TH Doan (https://thdoan.github.io/strftime/)


    Konverteringssekvensbeskrivningar

    Sekvens Beskrivning
    %a Veckodagens förkortade namn.
    %A Veckodagens fullständiga namn.
    %b Förkortat månadsnamn.
    %B Hela månadens namn.
    %c Representation av föredraget datum och tid (UTC) för den aktuella lokalen.
    %C Århundradetal (år/100) som ett tvåsiffrigt heltal.
    %d Dag i månaden som ett decimaltal (intervall 01 till 31).
    %e Dag i månaden som ett decimaltal (intervall 1 till 31).
    %F ISO 8601 datumformat (motsvarande %Y-%m-%d ).
    %G ISO 8601 veckobaserat årtal med sekel som decimaltal. Det 4-siffriga året motsvarar ISO-veckans nummer (se %V ). Detta har samma format och värde som %Y , förutom att om ISO-veckans nummer tillhör föregående eller nästa år, används det året istället.
    %g Som %G , men utan sekel, alltså med ett 2-siffrigt årtal (00-99).
    %H Timme som ett decimaltal med en 24-timmarsklocka (intervall 00 till 23). Se även %k .
    %I Timme som ett decimaltal med en 12-timmarsklocka (intervall 01 till 12). Se även %l .
    %j Årets dag som ett decimaltal (intervall 001 till 366).
    %k Timme som ett decimaltal med en 24-timmarsklocka (intervall 0 till 23). Se även %H .
    %l Timme som ett decimaltal med en 12-timmarsklocka (intervall 1 till 12). Se även %I .
    %m Månad som ett decimaltal (intervall 01 till 12).
    %n Månad som ett decimaltal (intervall 1 till 12).
    %M Minute som ett decimaltal (intervall 00 till 59).
    %p Antingen "AM" eller "PM" enligt det givna tidsvärdet. Middag behandlas som "PM" och midnatt som "AM".
    %P Som %p men med gemener ("am" eller "pm").
    %s Antal sekunder sedan epoken, 1970-01-01 00:00:00 +0000 (UTC).
    %S Andra som ett decimaltal (intervall 00 till 59).
    %u Veckodag som decimal (intervall 1 till 7), måndag är 1. Se även %w .
    %V ISO 8601 veckonummer för innevarande år som ett decimaltal (intervall 01 till 53), där vecka 1 är den första veckan som har minst 4 dagar på det nya året (det vill säga den första torsdagen).
    %w Veckodag som decimal (intervall 0 till 6), söndag är 0. Se även %u .
    %x Önskad datumrepresentation för den aktuella lokalen utan tiden.
    %X Önskad tidsrepresentation för den aktuella lokalen utan datum.
    %y År som ett decimaltal utan ett sekel (intervall 00 till 99).
    %Y År som ett decimaltal inklusive århundradet.
    %z Den numeriska tidszonen +hhmm eller -hhmm (det vill säga tim- och minutförskjutningen från UTC).
    %Z Tidszonens namn eller förkortning.
    %t Kort tidszon förkortning.
    %0 Bokstavligt år. Lokaliserad, främst för asiatiska platser. (zh-CN: 年, ko: 년)
    %1 Månad bokstavlig. Lokaliserad, främst för asiatiska platser. (zh-TW: 月, ko: 월)
    %2 Dag bokstavlig. Lokaliserad, främst för asiatiska platser. (ja: 日, ko: 일)

    Anmärkningar om kompatibilitet:

    • %c - formaterad sträng är något annorlunda
    • %D - inte implementerad (använd %m/%d/%y eller %d/%m/%y )
    • %e - mellanslag har inte lagts till
    • %E - inte implementerad
    • %h - inte implementerad (använd %b )
    • %k - blanksteg har inte lagts till
    • %n - som %m , men ingen inledande nolla (använd \n för nyrad)
    • %O - ej implementerad
    • %r - inte implementerad (använd %I:%M:%S %p )
    • %R - inte implementerad (använd %H:%M )
    • %t - inte implementerad (använd \t )
    • %T - inte implementerad (använd %H:%M:%S
    • %U - inte implementerat
    • %W - inte implementerat
    • %+ - inte implementerat
    • %% - inte implementerad (använd % )

    PDF-utdata och automatisk filnamngivning

    PDF-utdata kan nu både namnges automatiskt med hjälp av ett tokensystem och sparas i en fördefinierad PDF-utdatakatalog. Detta är särskilt användbart när du skriver ut flera utvalda e-postmeddelanden eftersom det inte är nödvändigt att välja namn och plats för varje e-postmeddelande.

    Aktivera PDF-utdatakatalogen och välj en katalog för lagringsplatsen. Utdatafilnamnet följer det konfigurerade filnamnsformatet.

    Filnamnsformatet består av tokens och eventuella statiska tecken som avgränsare eller blanksteg. Den utökade filnamnsformateringen använder tokens. Det exakta filnamnet kan konstrueras med hjälp av tokens såväl som vilken text eller giltiga filnamnstecken som helst. Ogiltiga tecken är: | * : < > ? \ / "

    Standardnamnformatet är: ${subject} som bara är ämnet för e-postmeddelandet.

    Exempel

    ${subject} – Standardformatet, ämnet för e-postmeddelandet
    ${prefix}_${date_custom}_${sender} – Använd det anpassade datumformatet istället för standard
    ${subject}-GmailMsg-${date}-${index} - Med lite anpassad text

    Tabellen nedan beskriver den fullständiga uppsättningen av tokens som kan användas i filnamnsformatet.

    Tecken Beskrivning
    ${subject} Ämnet för meddelandet.
    ${sender} Namnet på meddelandeavsändaren.
    ${recipient} Namnet på meddelandemottagaren.
    ${sender_email} E-postadressen till meddelandeavsändaren.
    ${recipient_email} E-postadressen till meddelandemottagaren.
    ${smart_name} Mottagarna om meddelandet finns i en Skickat-mapp, annars avsändarens namn
    ${index} Meddelandets indexnummer.
    ${prefix} Prefixet textalternativ värde.
    ${suffix} Alternativvärdet för suffixtexten.
    ${date} Standard datumformat. Standardformatet är: ÅÅÅÅMMDDHHmm
    ${date_custom} Det anpassade datumformatet. Se avsnittet om anpassad datumformatering.

    Utskrifts- och sidalternativ

    Dessa alternativ låter dig ställa in standardskrivaren och alternativen per skrivarsida. Om det beständiga läget är inställt för standardskrivaren, kommer en skrivarändring i förhandsgranskningen att återställas till PrintingTools NG-skrivaren för varje efterföljande utskrift. Det är också möjligt att ställa in det tysta läget där förhandsgranskningsdialogrutan inte visas och utskriften görs omedelbart.

    Sidalternativen låter dig ställa in objekt när både förhandsgranskning eller utskrift är klar. Dessa alternativ är beständiga över hela utskriften. Om alternativen som ställts in i PrintingTools NG ändras i förhandsgranskningen av utskriften kommer de inte att påverka efterföljande utskrifter. Vissa objekt är inte tillgängliga för att ställas in i förhandsgranskningsdialogrutan, dessa inkluderar sidhuvuden och sidfötter. Andra objekt behålls inte av förhandsgranskningen och skulle normalt behöva ställas in varje gång en utskrift görs. PrintingTools NG löser båda dessa problem.

    Ett objekt som inte sparas av förhandsgranskningen är sidintervallen. Ett typiskt användningsfall är när man vill skriva ut bara den första sidan i ett e-postmeddelande som standard. normalt skulle man behöva ändra sidintervallen varje gång. Observera att om ett sidintervall som ställts in i PrintingTools NG inte är giltigt, till exempel att överskrida antalet sidor, måste intervallet korrigeras i förhandsgranskningen innan utskrift. Ett sidintervall inställt på "1" kommer alltid att vara giltigt och skriver ut första sidan i meddelandet.

    Anmärkningar:

    Skalningsalternativet låter en ställa in en beständig skala.

    Sidintervall beskrivs på samma sätt som i förhandsgranskningen. Exempel är 1, 1-2, 3-4,5. Observera att om ett förinställt sidintervall inte är giltigt för ett givet meddelande kommer förhandsgranskningen att visa ett fel.

    Marginalerna kan ställas in med hjälp av enheter som är lämpliga för användarens lokalitet, antingen tum eller millimeter (in eller mm). Observera att på grund av avrundningsfel kan vissa mm-värden sparas med små skillnader.

    Sidhuvuden och sidfötter kan ställas in så att de inkluderar olika meddelandevariabler som anges nedan. Dessutom kan godtycklig text inkluderas. För att inte skriva ut sidhuvuden eller sidfötter och spara utskriftsutrymme, gör bara alla fält tomma.

  • &D = Datum
  • &P = Sidnummer
  • &PT = Sidnummer med totalt ("3 av 5")
  • &T = Sidrubrik (Ämne för meddelandet)
  • &U = URL (webbadressen till meddelandet)

    Felsökningsalternativ

    Felsökningsalternativen är en uppsättning nyckelord som anges som en blankstegsseparerad lista. Alternativen används vanligtvis för konsolfelsökning, men kan också användas för testning eller olika lägen.

  • initialsource - visa meddelandekällan innan PTNG gör någon formatering
  • finaloutput - visa det slutliga meddelandet
  • pdfoutput - debug output för PDF-meddelandeutdata
  • printsettings - felsök utdata för utskrifts- och sidalternativ
  • helpinwin - lägesflagga för att visa hjälp i ett fönster istället för standarden i en flik

PrintingTools NG ヘルプ
v3.0.0

PrintingTools NG 概要

PrintingTools NG はメッセージ印刷で多くのカスタマイズを提供します。 この拡張機能にはヘッダー情報とメッセージ本文の両方に多くの設定項目があります。 さらに通常の Thunderbird の印刷プレビューダイアログではできないページ設定も可能になります。 PDF 出力は柔軟なトークンベースのシステムを使用して自動命名を可能にします。

メールヘッダーツールバーへの印刷 NG ボタンを追加：

印刷 NG ボタンは Thunderbird 115 以降、統合ツールバーに自動的に追加されました。 このボタンはメールヘッダーツールバーにも追加することができ、メッセージペインから設定と印刷の両方へのアクセスも可能になります。 さらにアドオンタブの拡張機能のレンチボタンからも設定にアクセスできます。


次の手順に従ってメールヘッダーツールバーにボタンを配置します:

  • メールヘッダーツールバーの追加ボタンをクリックします。
  • [カスタマイズ...] を選択すると [ツールバーのカスタマイズ] ウィンドウが開きます。
  • [印刷 NG] ボタンを見つけます。
  • ボタンを目的の場所にドラッグアンドドロップします。
  • ボタンを使用して印刷、プレビュー、設定、およびヘルプを表示します。

    • PrintingTools NG のキーボードショートカット

    PrintingTools NG は標準の Thunderbird 印刷ショートカットを再マップし、標準の印刷の代わりに 自分自身を呼び出します。 Windows と Linux では Ctrl+P、Mac では command+P を使用します。 同様に Ctrl+Shift+P、command+shift+P で設定を開くことも可能です。

    以下はヘッダーの書式設定機能の一部の例です:

  • CSS を使用してカスタマイズしたヘッダーの枠線 (上下の青線)
  • ヘッダーの背景色
  • ヘッダーの行頭整列
  • カスタマイズしたヘッダーの並び替え
  • 任意のタイトルとメッセージ保存フォルダー名
  • ファイルアイコンが表示された添付リスト
  • 完全にカスタマイズ可能なロケール対応の日時形式


    • カスタム日時形式

    カスタム日時設定を使用するとメッセージヘッダーの日時と受信日時の両方に使用される書式を完全に制御できます。 カスタム日時にはほぼすべての形式を可能にする複数のローカライズされた設定を提供します。 カスタムの日時文字列には区切り記号、Unicode 文字を含む文字列、および以下にリストされている日付・時刻トークンを混在して含めることができます。

    注意: 以下の例のカスタム日時の結果は実際の日時、ユーザーのロケールおよびタイムゾーンによって異なります。

    例:

    日時文字列 結果 備考
    %Y%0%m%1%d%2 %a %t %H:%M:%S [GMT %z] 2022年04月20日 水 JST 16:06:45 [GMT +0900] 日本語(ja)ロケールの文字列
    %Y%0%m%1%d%2 %a %t %H:%M:%S 2022년04월20일 수 KST 16:11:06 韓国語(ko)ロケールの文字列
    %Y%0%m%1%d%2 %a %H:%M:%S %Z 2022年04月20日 周三 18:01:42 北京时间 中国語(zh-CN)ロケールの文字列
    %Y%0%m%1%d%2 %A %H:%M:%S %t 2022年04月20日 星期三 18:01:42 CST 中国語(zh-TW)ロケールの文字列
    %d.%m.%Y %H:%M:%S %t 20.04.2022 18:01:42 CET ドット区切りの文字列
    %a, %b %d %Y %H:%M:%S Mi, Apr 20 2022 18:01:42 ドイツ語(de)ロケールの文字列
    %A, %B %d %Y (%H:%M:%S) %t Wednesday, April 20 2022 (18:01:42) EDT 英語(en-US)ロケールの文字列

    カスタム日時形式は Unix のような機能 strftime() に一致する % シーケンスを利用します。これは日付と時刻をフォーマットする関数である strftime() を JavaScript に移植したものです。
    TH Doan による strftime() の移植版 (https://thdoan.github.io/strftime/)

    変換シーケンスの説明

    シーケンス 説明
    %a 曜日の略称。
    %A 曜日のフルネーム。
    %b 月の略称。
    %B 月のフルネーム。
    %c 現在のロケールの優先日時(UTC)表現。
    %C 2桁の整数の世紀番号(年/100)。
    %d 10進数の月の日(01から31の範囲)。
    %e 10進数の月の日(1から31の範囲)。
    %F ISO 8601 日付形式(%Y-%m-%d に相当)。
    %G 10進数で世紀を表示する ISO 8601 週ベースの年。4桁の年は ISO の週番号に対応します (%V を参照)。これは %Y と同じ形式と値ですが ISO 週番号が前年または翌年に属する場合は代わりにその年が使用されます。
    %g %G と同様ですが世紀はありません。つまり、2桁の年(00-99)です。
    %H 24時間制(00から23の範囲)を使用した10進数の時間。 %k も参照してください。
    %I 12時間制(01から12の範囲)を使用した10進数の時間。 %l も参照してください。
    %j 10進数の年の日(001から366の範囲)。
    %k 24時間制(0から23の範囲)を使用した10進数の時間。 %H も参照してください。
    %l 12時間制(1から12の範囲)を使用した10進数の時間。 %I も参照してください。
    %m 10進数の月(01から12の範囲)。
    %n 10進数の月(1から12 の範囲)。
    %M 10進数の分(00から59の範囲)。
    %p 指定された時間値に応じて「AM」または「PM」のいずれか。正午は「PM」、正子(午前0時)は「AM」として扱われます。
    %P %p と同様ですが小文字("am" または "pm")です。
    %s 1970-01-01 00:00:00 +0000(UTC) からのエポック秒数(Unix 時間)。
    %S 10進数の秒(00から59の範囲)。
    %u 月曜日を1とする10進数の曜日(1から7の範囲)。 %w も参照してください。
    %V 現在の年の ISO 8601 週番号を10進数で表したもの(01から53の範囲)。ここで週1は 新しい年の少なくとも4日を含む最初の週(つまり、最初の木曜日)です。
    %w 10進数の曜日(0から6の範囲)で日曜日は0です。 %u も参照してください。
    %x 時刻なしの現在のロケールの優先日付表現。
    %X 日付なしの現在のロケールの優先時間表現。
    %y 世紀のない10進数の年(00から99の範囲)。
    %Y 世紀を含む10進数の年。
    %z +hhmm または -hhmm の数値タイムゾーン (つまり、UTC からの時間と分のオフセット)。
    %Z タイムゾーン名または略称。
    %t タイムゾーンの略称。
    %0 年文字。主にアジアのロケール向けにローカライズされています。(zh-CN:年、ko:년)
    %1 月文字。主にアジアのロケール向けにローカライズされています。(zh-TW:月、ko:월)
    %2 日文字。主にアジアのロケール向けにローカライズされています。(ja:日、ko:일)

    互換性に関する注意事項:

    • %c - フォーマットされた文字列が少し異なります
    • %D - 実装されていません (%m/%d/%y または %d/%m/%y を使用)
    • %e - スペースは追加されません
    • %E - 実装されていません
    • %h - 実装されていません (%b を使用)
    • %k - スペースは追加されません
    • %n - %m と同様ですが、先行ゼロはありません (改行には \n を使用してください)
    • %O - 実装されていません
    • %r - 実装されていません (%I:%M:%S %p を使用)
    • %R - 実装されていません (%H:%M を使用)
    • %t - 実装されていません (\t を使用)
    • %T - 実装されていません (%H:%M:%S を使用)
    • %U - 実装されていません
    • %W - 実装されていません
    • %+ - 実装されていません
    • %% - 実装されていません (% を使用)

    PDF 出力とファイルの自動命名

    PDF 出力はトークンシステムを使用して自動で名前を付けることができ、あらかじめ設定された PDF 出力フォルダーに保存します。 この機能は選択した複数のメッセージを印刷する際にメッセージ毎に名前と場所を選択する必要がないため特に便利です。

    PDF 出力フォルダーを有効にして保存場所のフォルダーを選択します。出力ファイル名は設定されたファイル名形式に従います。

    ファイル名形式はトークンと区切り記号や空白文字などの静的な文字列から構成されます。拡張ファイル名形式はトークンを使用します。 トークンおよび任意のテキストや有効なファイル名文字列を使用して正確なファイル名を作成することができます。無効な文字は次のとおりです: | * : < > ? \ / "

    デフォルトの名前形式は ${subject} で、これはメッセージの件名です。

    例

    ${subject} - デフォルトの形式、メッセージの件名
    ${prefix}_${date_custom}_${sender} - デフォルトの代わりにカスタムの日時形式を使用
    ${subject}-GmailMsg-${date}-${index} - カスタムテキスト付き

    次の表はファイル名形式で使用できるトークン一式を示したものです。

    トークン 説明
    ${subject} メッセージの件名。
    ${sender} メッセージ送信者の名前。
    ${recipient} メッセージ受信者の名前。
    ${sender_email} メッセージ送信者のメールアドレス。
    ${recipient_email} メッセージ受信者のメールアドレス。
    ${smart_name} メッセージが送信済みトレイにある場合は受信者名、それ以外の場合は送信者名。
    ${index} メッセージのインデックス番号。
    ${prefix} 接頭語のテキスト設定値。
    ${suffix} 接尾語のテキスト設定値。
    ${date} 標準の日時形式。デフォルトの形式は YYYYMMDDHHmm です。
    ${date_custom} カスタムの日時形式。カスタム日時形式の項目を参照。


    プリンターとページ設定

    この機能を使用するとデフォルトで使うプリンターと印刷ページ毎の設定ができます。 印刷プレビューダイアログを表示せずに印刷するサイレントモードも設定できます。

    ページ設定では印刷プレビューダイアログ、または印刷時の両方の項目を設定することができます。これらの設定は印刷全体で保持します。 PrintingTools NG で設定された内容は印刷プレビューダイアログで変更しても、その後の印刷には影響しません。 印刷プレビューダイアログで設定できない項目にヘッダーとフッターがあります。他の項目も印刷プレビューダイアログでは保持されず、通常は 印刷をおこなう度に設定する必要があります。PrintingTools NG はこれらの問題を両方とも解決します。

    印刷プレビューダイアログで保存されない項目のひとつは ページ指定です。典型的な使用例はメッセージの最初のページだけを毎回印刷したい場合です。通常は毎回ページ指定を変更する必要があります。 もし PrintingTools NG で設定したページ指定が総ページ数を超えるなどで適用できない場合は印刷前に印刷プレビューダイアログでページ指定を 修正する必要があります。 ページ指定を "1" に設定すると常に有効になり、メッセージの最初のページを印刷します。

    注意事項：

    倍率設定を使用すると永続的な倍率を設定できます。

    ページ指定は印刷プレビューダイアログと同じように記述します。例としては 1, 1-2, 3-4,5 などです。 但し、設定したページ指定がメッセージに対して適用できない場合は印刷プレビューダイアログでエラーを表示します。

    余白はユーザーのロケールに適した単位 (インチまたはミリメートル (in または mm)) を使用して設定できます。四捨五入による誤差のため一部の mm 値はわずかな違いで保存される場合があります。

    ヘッダーとフッターは以下に示す様々なメッセージ変数を設定することができます。さらに任意のテキストを含めることも可能です。 ヘッダーやフッターを印刷せずに印刷スペースを節約するときはすべてのフィールドを空白にします。

  • &D = 日時
  • &P = ページ番号
  • &PT = ページ番号 / 総ページ数 (3/5)
  • &T = タイトル (メッセージの件名)
  • &U = URL (メッセージの URL)


    • 高度なオプション

    これらは基本的なUI印刷設定を超えて設定できるオプションです。 現在、ヘッダーとフッターの端からの余白を制御するための１つの高度なオプションがあります。 これは「全般設定」タブの「高度なオプション」で設定でき、スペースで区切ります。
    形式は次のとおりです:

    P:プリンター名::S:オプション名=値

    プリンター名のスペースはアンダースコアに置き換えられ、 標準内蔵の「PDF に保存」は Mozilla_Save_to_PDF です。

    別の例: Microsoft_Print_to_PDF

    現在のオプション名は次のとおりです。
    edges, edgeTop、edgeBottom、edgeLeft、edgeRight

    edges は、すべての端からの余白を同じ値に設定する疑似オプションです。 余白はインチ単位です。

    例:

    P:Mozilla_Save_to_PDF::S:edges=0.25
    P:Microsoft_Print_to_PDF::S:edgeTop=0.1

    edgeTop の余白を 0.25 インチにして、その他のすべての 余白を 0.10 インチにしたい場合は、次のように入力します:

    P:Mozilla_Save_to_PDF::S:edges=0.10 P:Mozilla_Save_to_PDF::S:edgeTop=0.25

    デバッグ設定

    デバッグ設定はスペースで区切ったリストとして入力する一連のキーワードです。この設定は通常はコンソールの デバッグ出力に使用されますがテストや別のモードにも使用できます。

  • initialsource - PrintingTools NG がフォーマットを行う前にメッセージのソースを表示します
  • finaloutput - 最終的なメッセージ出力を表示します
  • pdfoutput - PDF メッセージ出力のデバッグ出力です
  • printsettings - 印刷およびページ設定のデバッグ出力です
  • helpinwin - ヘルプを標準のタブではなくウィンドウで表示するモードフラグです
